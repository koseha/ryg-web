# Record Your Games (RYG) MVP - lovable 프롬프트 모음

## 1. 메인 레이아웃 & 네비게이션

```
Record Your Games (RYG) 웹 앱의 메인 레이아웃을 만들어주세요.

**중요: 모든 UI 텍스트는 한국어로 작성해주세요.**

요구사항:
- Next.js App Router 구조
- 상단에 네비게이션 바 (RYG 로고, home/universe/league 메뉴)
- 우측 상단에 사용자 프로필 드롭다운 (로그인/로그아웃)
- 반응형 디자인 (모바일/데스크톱 대응)
- 다크 테마
- 깔끔하고 모던한 디자인

네비게이션 메뉴:
- Home: 메인 랜딩 페이지
- Universe: 모든 리그 탐색
- League: 가입된 리그 목록 관리

사용자 프로필 드롭다운:
- 프로필 보기
- 로그아웃
```

## 2. 홈페이지 (랜딩)

```
Record Your Games (RYG) 홈페이지를 만들어주세요.

디자인:

- 기능 소개 카드 섹션
- 시작하기 버튼 → /universe/list로 이동
- 리그 오브 레전드 테마 색상 (금색, 파란색 계열)
```

## 3. Universe 리스트 페이지

```
공개 리그 목록을 보여주는 페이지를 만들어주세요.

**중요: 모든 UI 텍스트는 한국어로 작성해주세요.**

기능:
- 리그 카드 그리드 레이아웃
- 각 리그 카드에 표시할 정보:
  * 리그 이름
  * 설명 (2줄 말줄임)
  * 멤버 수 (예: "12명 참여중")
  * 생성일
  * 진행된 매치 수
  * 최근 매치 진행일
  * "자세히 보기" 버튼

상단 섹션:
- 페이지 제목: "모든 리그 둘러보기"
- 검색 바 (리그 이름으로 검색)
- 정렬 옵션 (최신순, 멤버 많은 순)

상태 처리:
- 로딩 상태 (스켈레톤 UI)
- 빈 상태 ("아직 생성된 리그가 없습니다")
- 검색 결과 없음 상태

디자인:
- 반응형: 모바일에서는 1열, 태블릿 2열, 데스크톱 3열 그리드
```

## 4. 리그 상세 페이지 (Universe)

```
공개 리그의 상세 정보를 보여주는 페이지를 만들어주세요.

**중요: 모든 UI 텍스트는 한국어로 작성해주세요.**

페이지 구조:
- 리그 헤더 (이름, 설명, 생성일, 책임자)
- 멤버 수 표시
- 리그 규칙 섹션
- 운영진 목록 (프로필 사진 + 닉네임)
- 막대 그래프 (티어 분포, 포지션 분포)
- 우측 사이드바: 가입 신청 카드(스크롤에 따라 이동)

가입 신청 카드:
- 리그 기본 정보 요약
- "가입 신청하기" 버튼 (가입 신청 모달 열기)
- 이미 가입된 경우: "이미 가입된 리그입니다"
- 대기중인 경우: "승인 대기중입니다"

가입 신청 모달:
- 티어 선택 (필수): 브론즈, 실버, 골드, 플래티넘, 에메랄드, 다이아, 마스터, 그랜드마스터, 챌린저
- 포지션 선택 (필수, 다중 선택): 탑, 정글, 미드, 원딜, 서포터
- 신청 메시지 (선택)
- 신청/취소 버튼

디자인:
- 카드 기반 레이아웃
- 멤버 아바타는 원형으로 표시
- 가입 신청 버튼은 눈에 띄는 색상
- 반응형 (모바일에서는 사이드바가 하단으로)
```

```
- 탭 네비게이션: Overview | Members | Matches | Settings

   - 매치 생성하기 (운영진 이상만)

- 카드 기반, 그리드 레이아웃
```

```
- 대기중인 신청 목록

멤버 목록 액션:

- 무한 스크롤링

책임자 권한:
- 역할 변경 드롭다운
- 멤버 추방 버튼 (확인 모달)

디자인:
- 반응형: 모바일에서는 카드 형태로 표시

```

```
- 매치 제목 입력
- 생성 버튼 클릭 시 6자리 랜덤 코드 생성
- 생성 완료 후 코드 표시 + 복사 기능

빈 상태:
- "아직 생성된 매치가 없습니다"
- "첫 매치를 생성해보세요" (운영진 이상만)

디자인:
- 매치 코드 복사 기능과 토스트 메시지 포함
```

## 9. 리그 설정 페이지

```
리그 설정 페이지를 만들어주세요. (운영진 이상만 접근)

**중요: 모든 UI 텍스트는 한국어로 작성해주세요.**

설정 섹션들:

1. 기본 정보:
   - 리그 이름 편집
   - 설명 편집 (textarea)
   - 규칙 편집 (textarea)
   - 저장 버튼

2. 권한 관리:
   - 운영진 목록
   - 운영진 추가/제거 (책임자만)
   - 책임자 위임 (책임자만, 확인 모달)

3. 위험 구역 (책임자만):
   - 리그 삭제
   - 빨간색 경고 스타일
   - 확인 모달 (리그 이름 입력하여 확인)

폼 검증:
- 필수 필드 체크
- 저장 성공/실패 토스트
- 변경사항 있을 때만 저장 버튼 활성화

권한별 접근 제어:
- 일반 멤버: 접근 불가 (404)
- 운영진: 기본 정보만 편집
- 책임자: 모든 권한
```

## 공통 컴포넌트 프롬프트

```
다음 공통 컴포넌트들을 만들어주세요:

**중요: 모든 UI 텍스트는 한국어로 작성해주세요.**

1. LoadingSpinner: 로딩 상태 표시
2. ConfirmModal: 확인/취소 모달
3. Toast: 성공/에러 메시지 토스트
4. EmptyState: 빈 상태 일러스트레이션
5. UserAvatar: 사용자 프로필 사진 컴포넌트
6. RoleBadge: 역할 표시 배지 (책임자/운영진/멤버)
7. TierBadge: 티어 표시 배지 (브론즈~챌린저)
8. PositionTags: 포지션 태그 컴포넌트 (탑, 정글, 미드, 원딜, 서포터)
9. CopyButton: 텍스트 복사 버튼 ("복사" 텍스트)
10. SearchInput: 검색 입력 필드 (placeholder: "검색...")
11. FilterDropdown: 필터링용 드롭다운 (역할별, 티어별, 포지션별)

모든 컴포넌트는:
- TypeScript 사용
- 반응형 디자인
- 다크 테마
- 모던한 디자인
- 눈에 띄는 색상은 적게 사용, 버튼 호버도 단순하게.
- 접근성 고려 (aria-label 등)
- 모든 텍스트는 한국어로 작성
```
